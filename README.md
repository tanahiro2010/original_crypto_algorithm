# オリジナル暗号化アルゴリズム開発

このプロジェクトは、オリジナルの暗号化アルゴリズムを開発することを目的としています。以下の手順で進めていきます。

## 1. 要件定義

- **セキュリティ要件**: アルゴリズムは現代の暗号化技術に対して安全である必要があります。
- **パフォーマンス要件**: アルゴリズムは効率的で、実用的な速度で動作する必要があります。
- **実装要件**: アルゴリズムは複数のプログラミング言語で実装可能である必要があります。また、**現在の暗号化アルゴリズムの基礎を使用して**開発される必要があります。
- **応用要件**: アルゴリズムは様々な用途（例: データ保護、通信の暗号化など）に適用可能である必要があります。


## 2. アルゴリズム設計

- **基本構造の設計**: アルゴリズムの基本的な構造を設計します。これには、暗号化と復号化のプロセス、キー管理の方法などが含まれます。
- **セキュリティ分析**: 設計したアルゴリズムのセキュリティを分析し、潜在的な脆弱性を特定します。
- **パフォーマンス分析**: アルゴリズムのパフォーマンスを分析し、必要に応じて最適化します。


### 2.1 基礎アルゴリズムの定義

- **暗号化アルゴリズム**: データを暗号化するためのアルゴリズムを定義します。
- **復号化アルゴリズム**: 暗号化されたデータを復号化するためのアルゴリズムを定義します。
- **キー管理アルゴリズム**: キーの生成、配布、管理の方法を定義します。

#### 2.1.1 使用する暗号化 / 演算 技術の選定

- XOR演算
- ビットシフト演算
- シーザー暗号
- AND / OR / NOT演算を使用した暗号化
- バイトシフト暗号
> [!IMPORTANT]
> これらの技術は、開発者が理解できるものを選択した結果です。
> 学習次第で新たな技術が追加される可能性があります。


## 脅威モデルと前提条件

- **目的:** 本セクションは、このプロジェクトで想定する攻撃者の能力および安全性に関する前提条件を明確にします。
- **想定攻撃者:** ネットワーク上で通信を傍受・記録できるパッシブな攻撃者、および通信内容を改竄できるアクティブな中間者（MITM）。ただし、鍵管理領域に直接侵入して秘密鍵を奪取する高度な物理攻撃者は想定外です。
- **攻撃者の能力:** 暗号文の収集、リプレイ、挿入、ビットフリップ等の改竄が可能。サイドチャネル（電力解析、タイミング攻撃）や物理的盗難は基本的に除外しますが、実装次第では脆弱になる点に注意してください。
- **セキュリティ前提:** 鍵は安全な乱数源で生成され、安全に保管されること。乱数生成器（CSPRNG）は十分に強力であること。鍵の再利用は行わないこと。
- **範囲外（Out of Scope）:** 物理攻撃、量子コンピュータによる攻撃耐性、社会工学的攻撃の防御は本プロジェクトの対象外とします。
- **セキュリティ目標:** 本アルゴリズムは機密性（暗号文から平文を復元困難にすること）と基本的な整合性（単純な改竄検知）を目指しますが、正式な暗号解析・第三者レビューを経ていないため、学習・実験目的に限定してください。
- **注意事項:** 実装や利用環境により安全性は大きく変わります。本プロジェクトの成果を本番環境で使用する前に、暗号の専門家によるレビュー・ペネトレーションテストを受けることを強く推奨します。
